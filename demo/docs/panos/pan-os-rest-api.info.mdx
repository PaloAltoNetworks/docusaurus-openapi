# PAN-OS REST API

<ApiLogo logo={{"url":"https://pan.dev/img/pan_dev_black.svg"}} darkLogo={{"url":"https://pan.dev/img/pan_dev_white.svg"}}></ApiLogo>

The REST API for PAN-OSÂ® (PA-VM,single-vsys-mode) enables you to interact remotely with firewalls.
To get started with the REST API, see
 the <a href="https://docs.paloaltonetworks.com/pan-os/10-0/pan-os-panorama-api/get-started-with-the-pan-os-rest-api/pan-os-rest-api.html">PAN-OS and Panorama API Guide</a>.
 The following is a simple example of REST API use.

1. To use the REST API, you must first generate the API key required for authenticating API requests. The following
 *cURL* example shows a *GET* request whose response will include the API key.

 ```shell
 curl -X GET 'https://YOUR_HOST_URL/api/?type=keygen&user=<username>&password=<password>'
 ```

The following shows the format of a successful response. The key element contains the API key to specify
as the value for the HTTP request header *X-PAN-KEY* in subsequent REST API requests.

 ```xml
  <response status = 'success'>
     <result>
         <key>YOUR_API_KEY</key>
     </result>
  </response>
```

2. The following *cURL* example shows a REST API request to return a list of predefined tags from the firewall:

```shell
  curl -X GET https://YOUR_HOST_URL/restapi/v10.0/objects/tags?location=predefined --header 'X-PAN-KEY: YOUR_API_KEY'
```

  The following is an example of a successful response:

```json
  {
    "@status": "success",
    "@code": "19",
    "result": {
      "@total-count": "2",
       "@count": "2",
      "entry": [
          {
            "@name": "Sanctioned",
            "@location": "predefined",
            "color": "color17"
          },
          {
            "@name": "empty",
            "@location": "predefined"
          }
       ]
    }
  }
```
